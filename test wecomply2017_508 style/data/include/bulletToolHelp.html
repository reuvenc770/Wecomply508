WeComply BulletTool

System Requirements

Since the WeComply authoring system runs only in Internet Explorer running on Windows, BulletTool has the same requirements. It may work in other browsers and on a Macintosh, but this has not been tested. A monitor resolution of at least 1024x768 is highly recommended. Lower resolutions will work, but scrolling will be required.

Getting Started

BulletTool is launched by clicking the Wrench icon below the PithyQuote box in the authoring system.

Hopefully most of BulletTool is self-explanatory, but if not, questions are welcome as are bug reports and suggestions for improvement.

Importing Text

Text can be Copied from Word or other text files, then Pasted into the Bullet Text box.

Creating Columns

To create two bullets lying side-by-side, create the bullets using "New Sub-bullet Below" so they are not siblings, then position the bullets. WFP does not wrap text to column width; it is up to the author to ensure that text does not overlap.

Exporting Your Work to the WeComply Authoring System

When your bullets are complete, click Export. Code for the bullets automatically be placed into the PithyQuote box of the chapter editor.

Using Terms

NOTE: Clicking on a term works only in Preview.

To specify a section of text as a term, select the text in the Bullet Text box, then click Term. You can choose from all terms which are defined in the chapter. If you do not see the term you want, go back to the authoring system, create the term, then return to BulletTool. Once you've selected a term, click Select, then click Submit. You can then preview the term using Preview.

To remove a term specification from a section of text, delete "<a href="term:x">" before the section of text, and "</a>" after it.

Keyboard Shortcuts

Pressing the TAB key is a shortcut for pressing the Submit button.

Using Color

IMPORTANT: Colorizing text is only available in the Vintage version of BulletTool. It is not available in the PDA version.

The default color for a bullet is the color set by the skin. In most cases this is black. To apply a different color to the selected bullet, use the color chip. To revert back to the default, click "Use Default".

Colors apply to an entire bullet. If you'd like to set the color of only a section of bullet text, click "Use Default", then place:

<font color="#xxxxxx"> before the section of text, and
</font> after it,

where xxxxxx is a color number in hexadecimal notation (e.g FF0000). For now, you can colorize a section of text only if the color for the entire bullet is set to default.

Text Sizing

The default font size for a bullet is the size set by the skin. In most cases this is 12pt. To apply a different size to the selected bullet, use the - and + size buttons. To revert back to the default, click "Use Default".

Sizing applies to an entire bullet. If you'd like to set the size of only a section of bullet text, place:

<span class="xxx"> before the section of text, and
</span> after it,

where xxx is one of the following:

    textMinus3
    textMinus2
    textMinus1
    textPlus1
    textPlus2
    textPlus3
    textPlus4
    textPlus5
    textPlus6
    textPlus7

The following are also available, for the PDA version of BulletTool only:

    textPlus11
    textPlus13
    textPlus15
    textPlus16

A Note about Absolute and Relative Font Size and Positioning

In general, a WFP skin determines the size of bullet fonts and spacing. For example, when you change bullet spacing using BulletTool, you are actually changing spacing relative to the default set by the skin. Font sizing, however, works a little differently. When a bullet size is set to default, the font size is determined by the skin. When it is increased or decreased from the default, however, relative values are not used. Instead, the font size is set to an absolute size.

At the time of the writing of this help, all skins in WFP use the same default font size and spacing, so none of this is really an issue. It will be important to keep these issues in mind, however, if a skin with a different default font size or spacing is ever created,.

Viewing the Generated Code

In most cases, you will not need to use this feature. However, if you'd like to view the bullet code BulletTool generates on Export, click the BulletTool title in the upper left of the BulletTool window. The code will appear at the bottom. (You may need to scroll down to see it.)
